# Интернационализация

KVT webapp использует i18next и react-i18next.

Setup следует официальному i18next TypeScript guide:

- [Getting started](https://www.i18next.com/overview/getting-started)
- [API](https://www.i18next.com/overview/api)
- [TypeScript](https://www.i18next.com/overview/typescript)
- [Plugins and utils](https://www.i18next.com/overview/plugins-and-utils)

## Расположение

Переводы лежат в app core layer:

```text
src/core/i18n/
  config.ts
  i18next.d.ts
  resources.ts
  locales/
    ru/common.ts
    ru/chat.ts
    en/common.ts
    en/chat.ts
```

`app` запускает i18n, но владение переводами не находится в `src/app`.

## Использование

Используй обычный react-i18next API:

```ts
const { t } = useTranslation('chat')

t('title')
```

Autocomplete работает потому, что `i18next.d.ts` регистрирует shape ресурсов.

## Добавление namespace

1. Добавь `ru/newNamespace.ts`.
2. Добавь `en/newNamespace.ts`.
3. Добавь namespace в `resources.ts`.
4. Используй `useTranslation('newNamespace')`.

## Tooling

VS Code настроен для [i18n Ally](https://github.com/lokalise/i18n-ally).

Recommended extension:

```text
lokalise.i18n-ally
```

Extension читает:

```text
app/webapp/src/core/i18n/locales/{locale}/{namespace}.ts
```

i18next docs также рекомендуют `i18next-cli` для extraction, sync, lint и type generation. Мы пока
не добавляли его, потому что текущий app держит переводы как typed TypeScript modules.
